Make sure, you have set the LSApplicationQueriesSchemes entries in the info.plist file, declaring which schemes it attempts to query. <key>LSApplicationQueriesSchemes</key> <array> <string>whatsapp</string> <string>instagram</string> </array> Do not include "://". If you do include "://" canOpenUrl: will return NO. When you make the actual call of canOpenUrl: you must still include the colon i.e. canOpenUrl:@"instagram:" You can find more information in this WWDC 2015 video. Also, this line is suspicious: var instagramURL: NSURL = NSURL(string: "instagram://user?username=USERNAME") I suspect you did copy this from Instagrams documentation. If so, replace USERNAMEwith your actual username. To isolate the error, you might want to try a simpler URL first: var instagramURL: NSURL = NSURL(string: "instagram://app")
