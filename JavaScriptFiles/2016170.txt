<p>So -- some of this is new to me and perhaps others will have valuable things to add to what I'm about to say...</p> <p>Maybe try something like this...</p> <pre><code>(function() { var epad = window.document.esCapture1 &amp;&amp; window.document.esCapture1.ConnectedDevice; alert(epad); !epad &amp;&amp; alert('Sorry epad is not Connected or drivers not installed'); })(); </code></pre> <p>In your example, I think some or all browsers may choke on the 1st line within the function if 'window.document.esCapture1' is itself undefined, so I've changed it to guard against this possibility.</p> <p>Also, putting this test in an anonymous function keeps the 'epad' variable out of the global namespace, which s a good idea (unless you need it elsewhere).</p> <p>The last line simply tests to see if the 'epad' variable is <em>falsy</em> -- meaning the 2nd alert will appear if 'epad' is null, undefined, or false.</p>
