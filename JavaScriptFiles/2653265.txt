<p>i have some <code>javascript</code> files included in my html file for months. It used to work just fine until couple of days ago, now my scripts don't run anymore and i get <code>javascript error "invalid XML attribute value &lt;script language=JavaScript&gt;\n"</code> with firebug.</p> <p>Does anyone know what this error means and how to get rid of it? i guess is something about that <code>newline</code> "\n" but i can't see that in my file if i open it.</p>
<p>IIRC, <code>language</code> was never part of any specification, at least not in XHTML (And I think not in HTML, either.)</p> <p>Try whether <code>&lt;script type="text/javascript"&gt;</code> works better.</p>
<p>Try to find that <code>&lt;script language=JavaScript&gt;</code> and replace with</p> <pre><code>&lt;script language="JavaScript"&gt; </code></pre> <p>or </p> <pre><code>&lt;script type="text/javascript"&gt; </code></pre>
<p>The error is because of missing quotes on <code>language=JavaScript</code>, but as the other answers say, it's best to replace it with <code>type="text/javascript"</code> (including the quotes!)</p> <p>Make sure to <em>always</em> put your attributes in quotes, in this case you would have gotten another validation error even after doing so, but in itself is still a validation error, more importantly, it's a validation error with very real side-effects in many cases.</p>
