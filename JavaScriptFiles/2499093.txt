How to create object of Date(&quot;23.03.2010&quot;)
<p>I have astring directly coming form the database and I am creating object of Date as<br> <code>Date dt=Date("23.03.2010")</code> and it is comin NaN </p> <p>whereas when I use <code>Date dt= Date("03/23/2010")</code> it works fine.</p> <p>Any Idea how I can get this working?.</p>
<p>The safest way if is you can return the date as milliseconds since 1970-01-01, then you can easily create a Date object from it. Example:</p> <pre><code>var n = 1269302400000; var dt = new Date(n); </code></pre>
<p>You must pass string (parsed) dates in MDY format. This is to prevent ambiguity (does 5/6/2010 mean 6th May or 5th June?)</p> <p>If you prefer, you can use <code>new Date(year, month, day)</code> format, and pass the arguments separately.</p>
<p>Note that you'll want to invoke <code>Date</code> with the <code>new</code> operator - <a href="https://developer.mozilla.org/en/Core_JavaScript_1.5_Reference/Objects/Date" rel="nofollow noreferrer">from the Mozilla Developer Center</a>:</p> <blockquote> <p>Invoking Date in a non-constructor context (i.e., without the new operator) will return a string representing the current time.</p> </blockquote> <p>The same page details the syntax of the <code>Date</code> constructor.</p> <p>If you are constructing a <code>Date</code> from a string the format accepted is governed by the rules of the <code>Date.parse</code> method. See <a href="http://msdn.microsoft.com/en-us/library/k4w173wk(VS.85).aspx" rel="nofollow noreferrer">Microsoft's <code>Date.parse</code> documentation</a> for a summary of these rules.</p>

You will have to subtract 1 from the parsed month value to get a correct date.</p> <pre><code>var dateString = "23.03.2010"; var dateParts = dateString.split("."); var dt = new Date(dateParts[2], dateParts[1] - 1, dateParts[0]); </code></pre>
