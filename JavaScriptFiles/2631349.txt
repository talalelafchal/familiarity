What does it mean when there is a number parameter passed to toString?
<p>I'm just wondering what it means to attach a number as a parameter to the <code>toString()</code> method</p> <p>E.g. <code>obj.toString(10);</code></p> <p>I googled and i have never seen a parameter before.</p>
<p>The additional parameter works only for <a href="https://developer.mozilla.org/en/Core_JavaScript_1.5_Reference/Global_Objects/Number/toString" rel="nofollow noreferrer"><code>Number.prototype.toString</code></a> to specify the radix (integer between 2 and 36 specifying the base to use for representing numeric values):</p> <pre><code>var number = 12345; number.toString(2) === "11000000111001" number.toString(3) === "121221020" // â€¦ number.toString(36) === "9ix" </code></pre>
<p>It's not defined as a globally-applicable argument to <code>toString</code>, it only makes sense on <code>Number</code>, where it specifies the base to write in. You can use eg. <code>n.toString(16)</code> to convert to hex.</p> <p>The other built-in objects don't use any arguments and JavaScript will silently ignore unused arguments, so passing <code>16</code> to any other <code>toString</code> method will make no difference. You can of course make your own <code>toString</code> methods where optional arguments can mean anything you like.</p>
<p>This only works on Number objects and is intended to give you a way of displaying a number with a certain radix:</p> <pre><code>var n = 256; var d = n.toString(10); // decimal: "256" var o = n.toString(8); // octal: "400" var d = n.toString(16); // hex: "100" var b = n.toString(2); // binary: "100000000" var w = n.toString(20); // base 20: "cg" </code></pre> <p>Note that the radix must be an integer between 2 and 36 or <code>toString()</code> will raise an error.</p>
