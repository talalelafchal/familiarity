TypeError when applying a textarea's value to a variable
<p><strong>JavaScript</strong></p> <pre><code>var textarea = document.getElementById("textarea").value; </code></pre> <p><strong>HTML</strong></p> <pre><code>&lt;textarea id="textarea" cols="100" rows="10"&gt;du hello!&lt;/textarea&gt; </code></pre>
<p>if you are using <strong>wysiwyg editor</strong> like <strong>tinymce</strong> or else. document.getElementById("textarea").value; will not work they have there own method to get data.</p> <pre><code>&lt;script type="text/javascript"&gt; function test(name){ var textarea = document.getElementById("textarea").value; alert(textarea) } &lt;/script&gt; &lt;textarea id="textarea" cols="100" rows="10"&gt;du hello!&lt;/textarea&gt; &lt;a href="javascript:void(0)" onclick="javascript:test('this is just testing')"&gt;Test TextArea&lt;/a&gt; </code></pre>
<blockquote> <p>TypeError: Result of expression <code>'document.getElementById("textarea")' [null]</code> is not an object.</p> </blockquote> <p>This error states that it couldn't find an element with the id "textarea". The result of this <code>getElementById</code> operation is therefore <code>null</code>, and you can't access <code>.value</code> on <code>null</code> because it doesn't have any properties.</p> <p>The root cause of this is that no element with the id "textarea" exists at the time of searching. Either you have a typo somewhere, or your script is running at a time when the element doesn't exist (yet). If your script is in the header, you'll want to write it like this:</p> <pre><code>document.ready = function () { ... script here ... } </code></pre> <p>That will delay the execution until the document is ready and all elements exist.</p>
