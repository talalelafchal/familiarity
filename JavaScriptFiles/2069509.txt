Implementing different CSS on page
<p>I want to implement different CSS style sheet using javascript or code behind on aspx page so that for different web browser my page look better. Can anyone have some solution about this problem? I try a lot to implement that but failed.</p>
<p>I'd go with the server side option (aspx in your case).</p> <ol> <li>check the '<a href="http://en.wikipedia.org/wiki/User_agent" rel="nofollow noreferrer">user_agent</a>' request header to determine the user's browser type</li> <li>logically include a different css file based on this variable</li> </ol>
<p>Generally you don't want to go down the route of dynamically generating CSS with Javascript. The best approach to CSS is to:</p> <ol> <li>Use a reset CSS;</li> <li>Declare a DOCTYPE on every page; and</li> <li>If necessary, include IE-specific additions (because, let's face it, it's always IE that causes the problems).</li> </ol>
<p>HtmlLink styleSheet = new HtmlLink(); styleSheet.Attributes.Add("rel","stylesheet"); styleSheet.Attributes.Add("type","text/css"); styleSheet.Attributes.Add("href",ResolveClientUrl("MyStyleSheet.css"));</p> <p>this.Page.Header.Controls.Add(styleSheet);</p> <p>Check this out.</p> <p>You can even set the style by adding a literal to your head tag and add the css style as text to this literal.</p> <p>Happy coding.</p>
<p>To add to Ravia: You can use Request.Browser to get browser versions:</p> <pre><code>HttpBrowserCapabilities bc = Request.Browser; if (bc.Browser == "IE" &amp;&amp; bc.Version == "6.0") { HtmlLink link = new HtmlLink(); link.Href = ResolveClientUrl("~/CSSFile.css"); link.Attributes.Add("rel", "stylesheet"); link.Attributes.Add("type", "text/css"); Page.Header.Controls.Add(link); } </code></pre>
