(by adding three zeros).</p> <p>Problem is, the redirect is made right away, without any delay at all... why?</p> <p>Thanks</p> <p>BTW its called like this: <code>&lt;body onload="redir();"&gt;</code></p>
<p>Try</p> <pre><code>setTimeout(function(){window.location = '/SV/main/main.html';}, 10); </code></pre> <p>10 is milliseconds.</p>
<p>You have to put your javascript statement between quotes :</p> <pre><code>function redir(){ setTimeout("window.location = '/SV/main/main.html';", 10); } </code></pre> <p>The delay is in milliseconds btw.</p> <p>As said in the comments and other answers, it is much cleaner to use an anonymous function to do such things :</p> <pre><code>function redir() { setTimeout(function(){ window.location = "/SV/main/main.html"; }, 10); // 10 milliseconds } </code></pre>
<p><em>setTimeout</em> takes a function and a timeout interval.</p> <pre><code>function redir() { setTimeout(function() { window.location = '/SV/main/main.html'; }, 10); } </code></pre>
<p>You should use the <code>href</code> property of location, and wrap the code within a function.</p> <pre><code>window.setTimeout( function() { window.location.href = '/SV/main/main.html';} , 10 ); </code></pre>
<p>Much cleaner way to write this:</p> <pre><code>function redir() { setTimeout(function(){ window.location = "/SV/main/main.html"; }, 10000); // fire after 10 seconds } </code></pre>
