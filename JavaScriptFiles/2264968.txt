Why is javascript:void(0) considered harmful?
<p>Can anybody tell me or point me to some resource which explains why using <code>javascript:void(0)</code> in hyperlinks is harmful (especially in Internet Explorer 6)?</p>
<p>Use of the <code>javascript:</code> keyword in a link isn't recommended anyway. I've only managed to find one article on why it might be harmful:</p> <p><a href="http://blog.reindel.com/2006/08/11/a-hrefjavascriptvoid0-avoid-the-void/" rel="nofollow noreferrer">a href=”javascript:void(0);” — avoid the void</a></p> <p>But the general consensus shows that you shouldn't use it because it might confuse browsers without javascript support, for some of those browsers it could be parsed as an invalid link.</p> <p>Instead, you should provide a link to a page either working around the functionality that would be provided by javascript or displaying a message about the site requiring javascript to work correctly. On the same link, <code>return false;</code> from your event, like so:</p> <pre><code>&lt;a href="noscript.html" onclick="doSomething(); return false;"&gt;I'm a link&lt;/a&gt; </code></pre> <p>Or alternatively, use <code>return false;</code> or <code>preventDefault()</code> and <code>returnValue</code> in your javascript code:</p> <pre><code>element.onclick = function () { /* // return false is better for most situations (see bobince's comment) if (event.preventDefault) event.preventDefault(); else event.returnValue = false; */ doSomething(); return false; } </code></pre>
<p>Click on <code>&lt;a href="javascript:void(0)" /&gt;</code> triggers event "beforeunload" in object "window" в IE (I have tested in IE10), but click on <code>&lt;a href="#" /&gt;</code> doesn't.</p>
