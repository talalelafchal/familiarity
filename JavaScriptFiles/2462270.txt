<p><strong>Performance Tip</strong> (<em>note</em> &mdash; this advice is valid for IE6, but it's no longer worth worrying about in 2014)</p> <p>If you're writing some Javascript code to build up a long string (say, a fairly big block of HTML, or a long parameter list for an ajax request), then <strong>don't get in the habit of doing this:</strong></p> <pre><code>var longString = ""; for (var i = 0; i &lt; someBigNumber; ++i) { if (i &gt; 0) longString += "&lt;br&gt;" + whatever; longString += someMoreStuff(); } </code></pre> <p>As the <code>longString</code> gets longer and longer, Internet Explorer will puff harder and harder on each iteration of the loop. Even when <code>someBigNumber</code> isn't really that big, the performance of that loop can be really terrible.</p> <p>Luckily, there's an easy alternative: <strong>use an array:</strong></p> <pre><code>var accumulator = []; for (var i = 0; i &lt; someBigNumber; ++i) { accumulator.push(someMoreStuff()); } var longString = accumulator.join("&lt;br&gt;" + whatever); </code></pre> <p><strong>Way, way faster</strong> in Internet Explorer than repeated string appends.</p>
