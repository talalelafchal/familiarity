What is 'click()' in JavaScript
<p>I was playing around with JavaScript in FireFox, and hit a problem that is illustrated in the following example:</p> <pre><code>&lt;HEAD&gt; &lt;script type="text/javascript"&gt; function click() { alert("click"); } &lt;/script&gt; &lt;/HEAD&gt; &lt;BODY&gt; &lt;input type="radio" onclick="click()"&gt; &lt;/BODY&gt; </code></pre> <p>When I click on the radio button, nothing happens, and no errors (in Firebug)</p> <p>If I change the name of the function to do_click, and change the onclick, then I get the alert.</p> <p>So question is: what is happening? click does not appear to be a reserved word, or an existing symbol</p>
<p>I think click() is a reserved procedure name, so you can't overwrite it. Explains why changing the name and nothing else gets it working.</p>
<p>Code within inline event handlers is scoped to the element, as if it was in a <code>with</code> block.</p> <p>Therefore, within the <code>onclick</code> handler, the name <code>click</code> resolves to the element's <code>click</code> method.</p> <p>This can be <a href="http://jsbin.com/ugilu" rel="nofollow noreferrer">demonstrated</a> by writing <code>alert(nodeName)</code> in the handler</p>
<p>I think the click() method in javascript simulates clicking the button. So this must be a sort of infinite loop...</p> <p>Edited to add: Check out <a href="http://msdn.microsoft.com/en-us/library/aa752277(VS.85).aspx" rel="nofollow noreferrer">MSDN</a> for documentation of the click method</p>
<p>DOM elements have a native <a href="https://developer.mozilla.org/en/DOM/element.click" rel="nofollow noreferrer" title="click()">click()</a> method.</p> <p>The following will show that click is a native method:</p> <pre><code>&lt;input type="radio" onclick="alert(click.toString())"&gt; </code></pre> <p>You can register your click function as the event handler as follows. Within the handler, <code>this</code> will refer to the HTML element.</p> <pre><code>&lt;input type="radio" id="foo" &gt; function click() { alert("click"); } document.getElementById('foo').onclick = click; </code></pre> <p>There is an excellent series of articles about browser events at <a href="http://www.quirksmode.org/js/introevents.html" rel="nofollow noreferrer">http://www.quirksmode.org/js/introevents.html</a></p>
<p>'click' by itself is NOT a javascript <a href="https://developer.mozilla.org/En/Core_JavaScript_1.5_Reference/Reserved_Words" rel="nofollow noreferrer">reserved keyword</a> outside the context of the inline event handler as Slaks answered.</p> <p>That is to say the "click" method defined in the question is valid. Attaching the event handler 'externally' will work. E.g. using jQuery:</p> <pre><code>$('input').click(click); </code></pre>
