window.location.replace JS
<p>What does the following code return?</p> <pre><code>window.location.replace("/ak012/(S(sar23pq1ki5wo22qqmmidvie))/HTML/Page.ashx?ID=4") </code></pre>
<p>In javascript, it changes the browser's location (redirects) to the url specified, <b>without</b> adding it to the browser history. Introduced in Javascript 1.1. <a href="http://docs.sun.com/source/816-6408-10/location.htm#1194240" rel="nofollow noreferrer">Documentation here</a>.</p>
<p><code>"/ak012/(S(sar23pq1ki5wo22qqmmidvie))/HTML/Page.ashx?ID=4"</code> - is a URL string <code>window.location.replace(URL_STRING)</code> - redirects to that URL</p>
<p>The replace() loads the specified URL over the current history entry. So when you use the replace method, the user cannot navigate to the previous URL by using Navigator's Back button.</p> <p>The argument inside the replace is your url : </p> <pre><code>/ak012/(S(sar23pq1ki5wo22qqmmidvie))/HTML/Page.ashx?ID=4 </code></pre>
<p>As Matt says, it returns undefined. It is a method for navigating to another document.</p> <p>I just want to add to the answers here a warning that IE does not register a referrer when navigating using the document.location properties and methods.</p> <p>My solution to this was to create a hidden A element in the document and programatically click that link.</p> <p>Example HMTL:</p> <pre><code>&lt;a href="http://www.stackoverflow.com" id="hidden-link" style="display: none"&gt;&lt;!-- Blank --&gt;&lt;/a&gt; </code></pre> <p>Example JavaScript:</p> <pre><code>var hiddenAElement = document.getElementById('hidden-link'); if ( document.all ) // Very simple IE detection. { hiddenAElement.click(); } else { document.location.href = hiddenAElement.href; } </code></pre>
