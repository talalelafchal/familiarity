<p>At the moment, you're probably best off using a library like <a href="http://prototypejs.org" rel="nofollow noreferrer">Prototype</a>, <a href="http://jquery.com" rel="nofollow noreferrer">jQuery</a>, <a href="http://code.google.com/closure/library" rel="nofollow noreferrer">Closure</a>, or similar for this sort of thing, as standardization for finding elements by class name (the <code>document.getElementsByClassName</code> method) is relatively recent and it's not supported yet by at least one major browser (they <a href="http://msdn.microsoft.com/en-us/library/dd282900(VS.85).aspx" rel="nofollow noreferrer">document a way</a> to add it, but not a very good one).</p> <p>In Prototype, the <code>$$</code> function searches by a large subset of <a href="http://www.w3.org/TR/2009/PR-css3-selectors-20091215/" rel="nofollow noreferrer">the CSS3 selectors draft</a> proposed recommendation. In jQuery, the <code>$</code> function does much the same thing (via the <a href="http://sizzlejs.com/" rel="nofollow noreferrer">Sizzle</a> selector engine, which can also be used on its own, weparate from jQuery).</p> <p>In Prototype, you can show or hide elements by class like so:</p> <pre><code>$$('div.myClassName').invoke('hide'); // Hides all matches $$('div.myClassName').invoke('show'); // Shows all matches </code></pre> <p>In jQuery, it's:</p> <pre><code>$('div.myClassName').hide(); // Hides all matches $('div.myClassName').show(); // Shows all matches </code></pre>
