for loop in hash table printing only last value
<p>Please advice.</p> <pre><code>for my $record (@item) { for my $int (@$record){ # DEBUG( "DEBUG:: $record and $int"); my %data = ( $record , $int ); } } } </code></pre> <p>Record is like </p> <pre><code>abc ,china abc ,japan abc , italy abc , singapore print Dumper %data; output : abc , singapore </code></pre> <p>Now the issues is when I dump the output it shows me last record entry in hash table.May be because of unique key.</p> <p>Kindly suggest.</p>
<p>Two problems: </p> <ol> <li><p>You are recreating the hash in each iteration of the loop. The correct way would be</p> <pre><code>my %data; for my $record (@item) { for my $int (@$record){ $data{$record} = $int; } } </code></pre></li> <li><p>Hash keys must be unique. It's not possible to have a hash like</p> <pre><code>( abc =&gt; 'china', abc =&gt; 'japan' ) </code></pre> <p>You can use a hash of arrays, though. Just assign to it with</p> <pre><code>push @{ $data{$record} }, $int; </code></pre> <p>It will create the following structure:</p> <pre><code>( abc =&gt; [ 'china', 'japan', 'italy', 'singapore' ] ) </code></pre></li> </ol>
