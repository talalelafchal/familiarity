Term::ReadLine: return values of readline
<p>To get the right output in the following script I have to decode the value returned by <code>readline</code>.</p> <p>Is it safe to assume that decoding the return-value of <code>readline</code> is required for all <a href="https://metacpan.org/release/Term-ReadLine" rel="nofollow">Term::ReadLine</a> backends? </p> <pre><code>#!/usr/bin/env perl use warnings; use strict; use 5.10.0; use Encode; use Encode::Locale; use Term::ReadLine; binmode STDOUT, ':encoding(console_out)'; my $term = Term::ReadLine-&gt;new( 'readline' ); say $term-&gt;ReadLine; # Term::ReadLine::Gnu my $line = $term-&gt;readline( ':' ); say decode( 'console_in', $line ); </code></pre>
