Mysql updating timestamp with curdate
<p>I'm working with Perl and Mysql, I testing this query in Navicat </p> <pre><code>UPDATE table SET times = ADDDATE(CURDATE(), INTERVAL 1 DAY) WHERE id = 1; </code></pre> <p>Example, when I exec in Navicat, I get result 2014-06-18 00:00:00 When I exec this query from perl script, I get 2014-06-18 15:22:12 Why in perl script not 00:00:00 ? column times have type timestamp.</p>
<p>I think the perl connector (DBI) is adding in the current time (Hh:mm:ss), whereas Navicat's connector does not. My suggestion would be to use <strong>NOW()</strong> instead of <strong>CURDATE()</strong>, which should solve this problem.</p> <p><img src="https://i.stack.imgur.com/bGExh.png" alt="MySQL Dates"></p> <p>Alternatively if you want it to be 00:00:00, then try this:</p> <pre><code>DATE_FORMAT(ADDDATE(CURDATE(), INTERVAL 1 DAY), '%Y-%m-%d') DATEONLY </code></pre>
