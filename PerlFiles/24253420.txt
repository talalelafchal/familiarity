Perl GetOpt::Long module error
<p>I am using GetOpt::long module to take the arguments from the command line and assign it to the respective variables. But I am getting errors when I am printing it. The code and the error is as follows:</p> <pre><code>#!usr/bin/perl use strict; use warnings; use Getopt::Long; GetOptions( "mount_path=s" =&gt; \my $old_path, "var=s" =&gt; \my $var, "log_path=s" =&gt; \my $log_path, ) or die "Error in input variables\n"; print &lt;&lt;"END_INPUTS"; These are your inputs: old_path= $old_path var = $var log_path=$log_path Press enter twice if all looks GOOD ********************************************************* END_INPUTS </code></pre> <p>The command line arguments are as follows:</p> <pre><code> perl getvar.pl --mount_path=/var/sslvpn --var=7.0.0.2_va-SSLVPN-!7.0.0.2.16sv+.jpn-05!j+g_554863- --log_path=log.txt </code></pre> <p>I am getting the following error while running this </p> <pre><code>-bash: !7: event not found </code></pre> <p>Please can someone help me with this. Thank you.</p>
<p>This is not a Perl problem. The bash shell is processing <code>!</code> as a special character. You'll have to quote that argument. </p> <pre><code> --var='7.0.0.2_va-SSLVPN-!7.0.0.2.16sv+.jpn-05!j+g_554863-' </code></pre> <p>You can tell that it's a bash problem and not a Perl problem because the message says it's from bash:</p> <pre><code>-bash: !7: event not found </code></pre> <p>Bash never even gets to the part where it runs your program.</p>
