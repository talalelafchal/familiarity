<p>When you interpolate an array, the elements are separated by the <code>$"</code> global variable. </p> <p>To avoid that behavior, just <a href="http://perldoc.perl.org/functions/print.html" rel="nofollow"><code>print</code></a> the array directly.</p> <pre><code> print @mkh; </code></pre> <p>This is documented in <a href="http://perldoc.perl.org/perlvar.html" rel="nofollow"><code>perlvar</code></a>:</p> <blockquote> <ul> <li><code>$LIST_SEPARATOR</code></li> <li><p><code>$"</code></p> <p>When an array or an array slice is interpolated into a double-quoted string or a similar context such as <code>/.../</code> , its elements are separated by this value. Default is a space. For example, this:</p> <pre><code> print "The array is: @array\n"; </code></pre> <p>is equivalent to this:</p> <pre><code> print "The array is: " . join($", @array) . "\n"; </code></pre> <p>Mnemonic: works in double-quoted context.</p></li> </ul> </blockquote> <p>On a side note, perhaps you'd like to be introduced to the <a href="http://perldoc.perl.org/functions/uc.html" rel="nofollow"><code>uc</code></a> and <a href="http://perldoc.perl.org/functions/lc.html" rel="nofollow"><code>lc</code></a> functions.</p> <pre><code>use warnings; use strict; while (&lt;DATA&gt;) { s/\s//g; print uc, "\n"; } __DATA__ jj dd ee aa ii hh kk ll gg bb cc ff </code></pre> <p>Outputs:</p> <pre><code>JJDDEEAAIIHHKKLLGGBBCCFF </code></pre>
