Hash notation in `%oneobs= %{$obs[1]};`
<p>In the documentation for <code>Statistics::Regression</code>, the instructions to add an observation is: $reg->include( 2.0, [ 1.0, 3.0, -1.0 ] );</p> <p>Then, to just run the regression on your own data:</p> <pre><code>my @obs; observations are like: %oneobs= %{$obs[1]}; ... foreach my $obshashptr (@obs) { $reg-&gt;include( $_[1], $_[3] ); } </code></pre> <p>I'm confused by <code>%oneobs= %{$obs[1]}</code>: what does this notation mean? Further, what is <code>foreach my $obshashptr (@obs) { $reg-&gt;include( $_[1], $_[3] ); }</code> doing? Could someone help by showing me what these two statement are doing in more than one line?</p> <p>It seems that <code>$_[1]</code> is the observation and <code>$_[3]</code> contains the predictor variables. So, <code>$_[3]</code> is an array..</p>
<p><code>my @obs;</code> is an array which contains hash reference at each index. </p> <pre><code>run_regression("bivariate regression", $obshashptr-&gt;{someY}, [ "const", "someX" ] ); </code></pre> <p>This calls the <code>run_regression</code> subroutine and passes 3 arguments.</p> <p>Now check the subroutine.</p> <pre><code>sub run_regression { my $reg = Statistics::Regression-&gt;new( $_[0], $_[2] ); foreach my $obshashptr (@obs) { $reg-&gt;include( $_[1], $_[3] ); } $reg-&gt;print(); } </code></pre> <p>The subroutine takes the arguments in <code>@_</code> array (see <a href="http://stackoverflow.com/questions/4563485/what-is-the-meaning-of-in-perl">What is the meaning of @_ in Perl?</a>). Then you create <code>$reg</code> object of <code>Statistics::Regression</code> and you pass 2 arguments to <code>new</code> method. <code>$_[0]</code> means the first element of <code>@_</code> array which in this case is <code>bivariate regression</code>. <code>$_[2]</code> means the third argument.</p> <p>Then <code>foreach</code> line loops through the array and assigns value at array's index to <code>$obshashptr</code> and then for each iteration of array you are calling the <code>include</code> method of <code>Statistics::Regression</code> and you are passing 2 arguments to it. First one is the value at second index of <code>@_</code> and second argument is value at 4th index of <code>@_</code>. </p> <p><img src="https://i.stack.imgur.com/brhhn.png" alt="http://i.imgur.com/cCMv1OT.png"></p>
<p><code>%{ }</code> is a hash dereference; what's inside it is a reference to a hash, and the contents of that hash are being assigned to <code>%oneobs</code>.</p> <p>See <a href="http://perlmonks.org/?node=References+quick+reference" rel="nofollow">http://perlmonks.org/?node=References+quick+reference</a></p>
