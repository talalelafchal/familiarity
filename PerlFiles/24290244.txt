regular expressions in perl for extracting information
<p>How would I match any number of any characters between two specific words... I have a document with a block of text enclosed between 'begin parameters' and 'end parameters'. These two phrases are separated by a number of lines of text. So my text looks like this:</p> <pre><code>begin parameters &lt;lines of text here \n. end parameters </code></pre> <p>My current regular expression looks like this:</p> <pre><code>my $regex = "begin parameters[.*\n*]end parameters"; </code></pre> <p>However this is not matching. Does anybody have any suggestions?</p>
<p>The meta-character <code>.</code> loses its special properties inside of a character class.</p> <p>So <code>[.*\n*]</code> actually matches 0 or more literal periods or zero or more newlines. </p> <p>What you actual want is to match 0 or more of any character <em>and</em> 0 or more of a newline. Which you can represent in a non-capturing group:</p> <p><code>begin parameters(?:.|\n)*?end parameters</code></p>
<p>Use the <code>/s</code> switch so that the any character <code>.</code> will match new lines.</p> <p>I also suggest that you use non greedy matching by adding <code>?</code> to your quantifier.</p> <pre><code>use strict; use warnings; my $data = do {local $/; &lt;DATA&gt;}; if ($data =~ /begin parameters(.*?)end parameters/s) { print "'$1'"; } __DATA__ begin parameters &lt;lines of text here. end parameters </code></pre> <p>Outputs:</p> <pre><code>' &lt;lines of text here. ' </code></pre>
<p>Your current regular expression does not do what you may think, by placing those characters inside of a character class; it matches any character of: ( <code>.</code>, <code>*</code>, <code>\n</code>, <code>*</code> ) instead of actually matching what you want.</p> <p>You can use the <code>s</code> modifier forcing the dot <code>.</code> to match newline sequences. By placing a capturing group around what you want to extract, you can access that by using <code>$1</code></p> <pre><code>my $regex = qr/begin parameters(.*?)end parameters/s; my $string = do {local $/; &lt;DATA&gt;}; print $1 if $string =~ /$regex/; </code></pre> <p>See <a href="https://eval.in/164493" rel="nofollow"><strong>Demo</strong></a></p>
<p>Please try this :</p> <pre><code>Begin Parameters([\S\s]+?)EndParameters </code></pre> <p>Translation : This will look for any char who is a separator, or any char who is everything but a separator (so actually, it will look for any char) until it find "EndParameters". I hope it is what you expect.</p>
