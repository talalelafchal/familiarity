Don't understand why swift wants me to be explicit
<p>I am testing out swift and don't understand why the following lines don't work.</p> <pre><code> let tabBarController = UITabBarController() tabBarController.viewControllers = [MyViewController1(), MyTableViewController2()] tabBarController.tabBar.items[0].title = "First" tabBarController.tabBar.items[1].title = "Second" </code></pre> <p>The compiler tells me: <code>Could not find member 'title'</code></p> <p>If I split the arguments apart into multiple variables, then it works.</p> <pre><code> let tabBarController = UITabBarController() tabBarController.viewControllers = [MyViewController1(), MyTableViewController2()] let tabBar = tabBarController.tabBar let tabBarItem0 = tabBar.items[0] as UITabBarItem let tabBarItem1 = tabBar.items[1] as UITabBarItem tabBarItem0.title = "First" tabBarItem1.title = "Second" </code></pre> <p>Can someone explain to me why the first version doesn't work?</p> <p>Thanks</p>
<p>Like you realized, <code>tabBar.items</code> is an Array of <code>AnyObject</code>, which is essentially <code>id</code>. You have to downcast it like you did with <code>as UITabBarItem</code>.</p> <p>One better way to do this, which is recommended, is to downcast the entire array before you start accessing. So it would look like this:</p> <pre><code>let tabItems = tabBar.items as [UITabBarItem] tabItems[0].title = "First" //... </code></pre>
