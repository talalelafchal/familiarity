Importing sqlite3.h to a Swift project via a Obj-C bridging header doesn't work
<p>I have created a file called <code>BridgingHeader.h</code> and it contains only one line:</p> <pre><code>#import &lt;sqlite.h&gt; </code></pre> <p>And of course the framework <code>libsqlite3.dylib</code> is imported either.</p> <p>In my build settings I've also set the value of <strong>Objective-C Bridging Header</strong> to <code>&lt;ProjectName&gt;/Bridging-Header.h</code></p> <p>BUT&#8230; <br /> When I add <code>import sqlite3</code> to my Swift class it says that it cannot find the module <code>sqlite3</code>.</p>
<p>Thanks to @trojanfoe I am using now a library which is an <code>Objective-C</code> wrapper for the <code>sqlite3</code> framework. (<a href="https://github.com/ccgus/fmdb" rel="nofollow">FMDB</a>)</p> <p>Got it to work by importing the <code>ObjC</code> header in my <code>BridgingHeader.h</code> file and using the class without importing anything in the <code>Swift</code> class.</p> <p><sub>Conclusion: Only Objective-C files can be imported into the bridging header&#8230;</sub></p>
<p>For anyone who does want to use SQLite 3 directly, see this Gist:</p> <p><a href="https://gist.github.com/zgchurch/3bac5443f6b3f407fe65">https://gist.github.com/zgchurch/3bac5443f6b3f407fe65</a></p>
