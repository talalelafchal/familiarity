Swift class as argument to objc runtime
<p>When using the objc runtime for Swift introspection, I noted that the following is possible: </p> <pre><code>object_setClass(anObject, Birdy.classForCoder()); </code></pre> <p>. . . but not </p> <pre><code>object_setClass(anObject, Birdy); </code></pre> <p>Both Birdy.classForCoder() and Birdy are an instance of AnyClass, so why is it that only the former will propagate into the objc runtime? </p>
<p>Use <code>Birdy.self</code> to get the <code>AnyClass</code> metatype.</p> <p>Example:</p> <pre><code>object_setClass(anObject, Birdy.self) </code></pre> <p>If you plan on making your Swift classes available within Obj-C, you should mark them with the <code>@objc</code> annotation:</p> <pre><code>@objc class Birdy { func tweet() { println("Tweet tweet! #swift") } } </code></pre>
