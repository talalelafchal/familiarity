<p>I'm facing the error message:</p> <pre><code>"UIStoryboardSegue does not have a member named 'identifier'" </code></pre> <p>Here's the code causing the error</p> <pre><code>if (segue.identifier == "Load View") { // pass data to next view } </code></pre> <p>On Obj-C it's fine using like this:</p> <pre><code>if ([segue.identifier isEqualToString:@"Load View"]) { // pass data to next view } </code></pre> <p>What am I doing wrong?</p>
<p>I think the problem is you have to use the ! to unbundle identifier</p> <p>I have</p> <pre><code>override func prepareForSegue(segue: UIStoryboardSegue?, sender: AnyObject?) { if segue!.identifier == "Details" { let viewController:ViewController = segue!.destinationViewController as ViewController let indexPath = self.tableView.indexPathForSelectedRow() viewController.pinCode = self.exams[indexPath.row] } } </code></pre> <p>My understanding is that without the ! you just get a true or false value</p>
<p>This seems to be due to a problem in the <code>UITableViewController</code> subclass template. It comes with a version of the <code>prepareForSegue</code> method that would require you to unwrap the segue.</p> <p>Replace your current <code>prepareForSegue</code> function with:</p> <pre><code>override func prepareForSegue(segue: UIStoryboardSegue!, sender: AnyObject!) { if (segue.identifier == "Load View") { // pass data to next view } } </code></pre> <p>This version implicitly unwraps the parameters, so you should be fine.</p>
<pre><code>override func prepareForSegue(segue: UIStoryboardSegue?, sender: AnyObject?) { if(segue!.identifier){ var name = segue!.identifier; if (name.compare("Load View") == 0){ } } } </code></pre> <p>You can't compare the the identifier with == you have to use the compare() method </p>
