'Receiver (&lt;ViewController&gt;) has no segue with identifier 'addSegue'
<p>I have a navigation controller that has a segue links between them called "addSegue". When I click on the <code>tableView</code> cell though the app crashes and I get the error below:</p> <pre><code>Terminating app due to uncaught exception 'NSInvalidArgumentException', reason: 'Receiver (&lt;MSAddFriendsViewController: 0x98cc340&gt;) has no segue with identifier 'addSegue' </code></pre> <p>I don't think that I have any issues with my code. Here is the method in which I have the line <code>showSegueWithIdentifier</code>:</p> <pre><code>- (void)tableView:(UITableView *)tableView didSelectRowAtIndexPath:(NSIndexPath *)indexPath { NSMutableSet *selectedUsers = [NSMutableSet set]; [self.tableView deselectRowAtIndexPath:indexPath animated:NO]; UITableViewCell *cell = [tableView cellForRowAtIndexPath:indexPath]; cell.accessoryType = UITableViewCellAccessoryCheckmark; PFRelation *friendsRelation = [self.currentUser relationforKey:@"friendsRelation"]; PFUser *user = [self.allUsers objectAtIndex:indexPath.row]; [friendsRelation addObject:user]; [self.currentUser saveInBackgroundWithBlock:^(BOOL succeeded, NSError *error) { if (error) { NSLog(@"Error %@ %@", error, [error userInfo]); } }]; [self performSegueWithIdentifier:@"addSegue" sender:self]; } </code></pre> <p><a href="https://drive.google.com/file/d/0B1Of4oDADQCQbXJuT3J3Y3pQNFU/edit?usp=sharing">Here is a picture of my storyboard</a></p> <p><a href="https://drive.google.com/file/d/0B1Of4oDADQCQbXJuT3J3Y3pQNFU/edit?usp=sharing">Here is an updated picture of my storyboard</a></p>
<p>Hard to say for sure, but some other people have had similar problems:</p> <ul> <li><p>In <a href="http://stackoverflow.com/questions/9568034/nsinvalidargumentexception-reason-receiver-has-no-segue-with-identifier">this question</a>, the asker instantiated the storyboard with <code>init</code> instead of <code>instantiateViewControllerWithIdentifier</code> so the segue wasn't set up properly.</p></li> <li><p>In <a href="http://stackoverflow.com/questions/11874200/nsinvalidargumentexception-receiver-has-no-segue-with-identifier">this question</a>, it was just something weird going on internally with xcode and the simulator, and running Product->Clean helped.</p></li> <li><p>And of course, it's possible the segue name in the code doesn't match the segue name on the Storybord, but I'm guessing you've checked that a lot of times already!</p></li> </ul>
<p><img src="https://i.stack.imgur.com/r0uKW.png" alt="enter image description here"></p> <pre><code>use segue identifier in Push Method and give the proper connection </code></pre> <p>if you are using <code>Identifier</code>, then call this line where u need</p> <pre><code>[self performSegueWithIdentifier:@"identifierName" sender:self]; </code></pre> <p><strong>Swift 2.X</strong></p> <pre><code>self.performSegueWithIdentifier("identifierName", sender: self) </code></pre> <p><strong>Swift 3</strong></p> <pre><code>self.performSegue(withIdentifier: "identifierName", sender: self) </code></pre> <hr> <p>Regarding the new screen you have added that way. In that screen, when you're finished and want to remove it, it's just:</p> <pre><code>self.dismiss(animated: false, completion: nil) </code></pre>
<p>I had this same problem and actually my problem was that I was calling</p> <pre><code>WRONG: [self.navigationController performSegueWithIdentifier:@"ShowVerify" sender:self]; </code></pre> <p>instead of</p> <pre><code>CORRECT: [self performSegueWithIdentifier:@"ShowVerify" sender:self]; </code></pre> <p>so check that you are calling correct performSegueWithIdentifier method :)</p>
