Use Quickblox.framework in Swift application
<p>I'm trying to connect <a href="http://quickblox.com/developers/IOS-how-to-connect-Quickblox-framework" rel="nofollow">QuickBlox.framework</a> to the Swift application.</p> <p>As I know QuickBlox.framework is written on Objective-C. To use it in Swift application we should follow <a href="https://developer.apple.com/library/prerelease/ios/documentation/swift/conceptual/buildingcocoaapps/MixandMatch.html" rel="nofollow">this guide</a></p> <p>What I did is:</p> <ol> <li>Create bridge header <strong>demo-swift-Bridging-Header.h</strong></li> <li>Import QuickBlox here: <code>#import &lt;Quickblox/Quickblox.h&gt;</code></li> </ol> <p>and I got 28 errors:</p> <pre><code>&lt;unknown&gt;:0: error: ./Quickblox.framework/Headers/Core/Definitions/Consts.h:16: unknown type name 'NSString' ... &lt;unknown&gt;:0: error: ./Quickblox.framework/Headers/Core/Definitions/Delegates.h:43: expected a type ... </code></pre> <p>Any ideas? </p>
<p>You have to modify a bit your bridge file - add all needed frameworks</p> <pre><code>@import Foundation @import SystemConfiguration; @import MobileCoreServices; #import &lt;Quickblox/Quickblox.h&gt; </code></pre> <p>Checkout QuickBlox blog post about Swift <a href="http://quickblox.com/blog/2014/06/building-ios-apps-with-quickblox-and-swift/" rel="nofollow">http://quickblox.com/blog/2014/06/building-ios-apps-with-quickblox-and-swift/</a> </p>
