Matlab addpath, genpath does not work in Mac?
<p>I have a folder named <code>'/Users/me/myfolder'</code>, which contains many subfolders. Each of these subfolders have .m or data files, whose names are unique. I did <code>addpath(genpath(userpath))</code>, and did <code>load('../data/filename')</code> and <code>load('filename')</code>, but I got errors: <code>No such file or directory</code>. How can I execute a file or load a file using <code>load('../data/filename')</code> or <code>load('filename')</code> regardless of which subfolder I am in? </p>
<p>Your question is a little ambiguous. </p> <ol> <li><code>userpath</code> does not neccesarly point to <code>/Users/me/myfolder</code></li> </ol> <p>The <code>userpath</code> is "the first folder or folders in the list of folders returned by PATHDEF and thus affects the search path"</p> <ol start="2"> <li>is '../data/filename' located in one of the subdirectories of <code>/Users/me/myfolder</code>?</li> </ol> <p>If not, then it will not be on the search path and you can not load it because Matlab does not know where that file is!</p> <hr> <p>To correctly add <code>/Users/me/myfolder</code> and all its subfolders, perform the following:</p> <p><code>addpath(genpath('/Users/me/myfolder'))</code></p> <p>Then ensure that <code>filename</code> is on that search path, if not do the same thing and add the path to <code>filename</code> to your search path using <code>addpath()</code> </p>
<p>In my case <code>userpath</code> returns:</p> <pre><code>/Users/my_user/Documents/MATLAB: </code></pre> <p>But <code>genpath('/Users/my_user/Documents/MATLAB:')</code> returns an empty string. The problem is that you need to remove the colon, so, a possible solution maybe is:</p> <pre><code>tmp = userpath addpath(genpath(tmp(1,end-1))) </code></pre>
