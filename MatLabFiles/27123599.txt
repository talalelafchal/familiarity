Passing a 1xn array into a function handle instead of individual values for Matlab?
<p>I'm trying to find a way to pass a 1xn array into a function handler with n being the number of variables in the function so let's suppose I declare a handler as such: </p> <pre><code>U = @(x, y) x^2 + 2*y^2 </code></pre> <p>and plugged in:</p> <pre><code>U(1, 2) ans = 9 </code></pre> <p>Is there some way I can do something similar to this instead?</p> <pre><code>a = [1, 2] U(a) ans = 9 </code></pre>
<p>Yes you can;</p> <pre><code>U = @(x, y) x^2 + 2*y^2; a = {1, 2}; U(a{:}) </code></pre> <p>When you expand the contents of a cell <code>{:}</code> it expands as the separate values stored in the cell. This is different from <code>myMatrix(...)</code> or <code>myCell(...)</code> which both produces a subset of the original set (be it a cell or a matrix).</p>
