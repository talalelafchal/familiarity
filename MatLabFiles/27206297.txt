Dimension mismatch of matrix
<p>I am not able to save the value of <code>BB</code> in <code>Bv</code>.</p> <p>MATLAB returns this error:</p> <blockquote> <p>Subscripted assignment dimension mismatch.</p> </blockquote> <p>Please help me to do it.</p> <pre><code>X=[1 6 9 5; 6 36 54 30; 9 54 81 40; 5 30 40 25] [N1,dim1]=size(X) ; for i=1:N1 bb=X(i:end,1)*X(i,i:end); BB=bb(triu(true(size(bb)))) Bv(i,:)=BB(:); end </code></pre>
<p>As @Rashid suggests, use cell arrays instead of numeric arrays. The beauty of cell arrays is that it can store matrices of different type and size in 1 storage unit. It is much like a structure, but with indices to easily call entries.</p> <pre><code>X=[1 6 9 5; 6 36 54 30; 9 54 81 40; 5 30 40 25]; for ii=1:size(x,1) bb=X(ii:end,1)*X(ii,ii:end); BB=bb(triu(true(size(bb)))) Bv{ii,:}=BB(:); end </code></pre> <p>Note that I also changed your loop index to use <code>ii</code> as opposed to <code>i</code>, see <a href="http://stackoverflow.com/questions/14790740/using-i-and-j-as-variables-in-matlab">here</a>. <code>i</code> is the imaginary unit and to prevent errors it's better to not overwrite build-in functions.</p> <p>Just an example of how a cell array stores different data types and sizes:</p> <pre><code>A = magic(2); % 2x2 double B = uint8(magic(3)); % 3x3 uint8 C = 'hello world'; % string YourCell{1} = A; YourCell{2} = B; YourCell{3} = C; YourCell = [2x2 double] [3x3 uint8] 'hello world' </code></pre> <p>The same but now as a structure:</p> <pre><code>YourStruct.magic2double = A; YourStruct.magic3uint8 = B; YourStruct.MyString = C; YourStruct = magic2double: [2x2 double] magic3uint8: [3x3 uint8] MyString: 'hello world' </code></pre> <p>The cell and structure contain the same information, but for information in the cell you call <code>YourCell{ii}</code>, whilst in the structure you must call <code>YourStruct.variablename</code>. The cell can be accessed by indexing, the structure cannot. For the structure however you can use easy names to remember what you stored in each element, whilst that's impossible for the cell.</p>
