Octave plotting with FLTK: Subplots getting smaller and smaller..?
<p>I'm using Octave 3.8.1.</p> <p>Consider the following code:</p> <pre><code>I=4; J=6; for j=1:J for i=1:I g=subplot(I,J,(i-1)*J+j); imagesc(rand(20,20)); end end </code></pre> <p>This produces the following plot: <img src="https://i.stack.imgur.com/qrEYP.png" alt="strangely sized subplots"></p> <p>Why aren't the subplots equally sized? What do I need to do to get them equally sized?</p> <p>I'm using the default FLTK module for plotting.</p>
<p>I'm writing my comments as an answer for those experiencing similar problems. </p> <p>From the ArchLinux documentation: <a href="http://wiki.archlinux.org/index.php/Octave" rel="nofollow noreferrer">http://wiki.archlinux.org/index.php/Octave</a> - it says FLTK has some serious instability issues. Try changing your default plotting system to <code>gnuplot</code> instead. </p> <p>Do this in the command prompt: <code>graphics_toolkit("gnuplot");</code></p> <p>FWIW, I actually reconstructed your figure using FLTK on my end, and this is what I get:</p> <p><img src="https://i.stack.imgur.com/sc1oo.png" alt="enter image description here"></p> <hr> <p>So weird!... but the weirdness of FLTK may be platform dependent. I'm on Mac OS, so that may contribute why I'm not getting any weird results.</p> <p>Glad to have helped!</p>
