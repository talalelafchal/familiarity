How NOT to overwrite the .mat file when using scipy.io.savemat()?
<p>I have been using Python's <code>scipy.io.savemat()</code> to save my data in .mat format, efficient way to transfer data between the two: <a href="http://docs.scipy.org/doc/scipy-0.9.0/reference/tutorial/io.html" rel="nofollow">http://docs.scipy.org/doc/scipy-0.9.0/reference/tutorial/io.html</a>.</p> <p>Somehow it overwrite the file. How to set the function so I can append to the .mat file instead of overwriting just like MATLAB save()'s '-append' option? Or is there any ways to work it out other than scipy.io.savemat()?</p>
<p>There doesn't appear to be a way to append with <code>scipy.io.savemat()</code>.</p> <p>You can <code>scipy.io.loadmat()</code>, append your new values to the dict that is returned, and then call <code>scipy.io.savemat()</code> again.</p> <p>Here is the documentation for <code>scipy.io.loadmat()</code>:</p> <p><a href="http://docs.scipy.org/doc/scipy-0.14.0/reference/generated/scipy.io.loadmat.html" rel="nofollow">http://docs.scipy.org/doc/scipy-0.14.0/reference/generated/scipy.io.loadmat.html</a></p>
